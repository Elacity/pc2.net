# Clawdbot Integration UI/UX Plan

> **Document**: UI/UX Design Specification for Clawdbot Integration  
> **Status**: Draft  
> **Created**: 2026-01-26  
> **Scope**: Multi-channel messaging integration with zero breaking changes to existing PC2 functionality

---

## Executive Summary

This document details how Clawdbot's multi-channel messaging capabilities integrate into PC2's existing UI/UX. The core principle is **additive enhancement** - all existing functionality remains unchanged while new features extend the AI Assistant experience.

---

## Guiding Principles

1. **Zero Breaking Changes**: Every existing feature works exactly as before
2. **Progressive Disclosure**: New features are discoverable but not intrusive
3. **Consistent Design Language**: Uses existing PC2 styling patterns
4. **Sovereign First**: All configuration emphasizes user control and privacy

---

### Quick Training Guide

**Step 1: Create AGENTS.md**
This file tells the agent how to behave. Include:
- What the agent's purpose is
- What topics it should/shouldn't discuss
- How to respond to questions
- Where to find information

**Step 2: Add Documentation**
Put your docs in the workspace. The agent uses the `read` tool to access them.
Organize logically so the agent can find relevant content.

**Step 3: Create Custom Skills (Optional)**
For specialized behaviors, create a `SKILL.md` in the skills folder.
Skills teach the agent specific procedures.

**Step 4: Test & Iterate**
Message your agent, see how it responds, refine AGENTS.md as needed.

## Implementation Phases

### Phase 1: Settings UI (1 week)
- Add Messaging Channels section to AI Settings tab
- Create WhatsApp QR code connection modal
- Create Telegram bot token input modal
- Channel status indicators

### Phase 2: Gateway Integration (1 week)
- Embed Clawdbot Gateway in PC2 node
- Route messages to existing AIChatService
- Add channel context to conversation storage

### Phase 3: AI Chat Enhancement (3 days)
- Add channel tabs to AI Chat header
- Show message source (Desktop/WhatsApp/etc)
- Unified conversation view

### Phase 4: Transaction Flow (3 days)
- Transaction proposals via messaging
- Approval commands in chat
- Desktop notification fallback

### Phase 5: Polish (3 days)
- Taskbar channel status
- Quick status popup
- Error handling and reconnection UI

---

## Success Criteria

1. **Zero Regression**: All existing AI Chat functionality works unchanged
2. **Discoverable**: New users can find and connect channels within 2 minutes
3. **Secure by Default**: Pairing mode prevents unauthorized access
4. **Unified Experience**: Conversations feel continuous across channels
5. **Sovereign**: All data and credentials remain on user's PC2 node
